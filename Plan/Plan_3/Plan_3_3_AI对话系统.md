# Plan_3_3：AI 对话系统实装

## 问题描述

### P1：AI 对话完全不可用（图五）
**现象**：
- 右侧对话面板显示「请先配置 API Key...」提示
- 即使配置了 API Key，发送消息后无法真正与 AI 通信
- AI 无法使用已注册的 Agentic 工具（view_file, list_dir, write_to_file 等虽显示在 Context 面板但不可执行）
- 底部状态栏显示「未配置」

**根因**：AgentLoop → AIClient → OpenAI API 的 streaming 管线虽然代码存在，但可能存在 API Key 传递断链、错误吞没等问题导致前端看不到任何响应。

### P2：文件/图片上传不可用（图六-第二张）
**现象**：
- 点击附件按钮后弹出"已选择 1 个文件 (附件功能完善中)"的 toast 提示
- 没有文件缩略图展示
- 文件内容未真实嵌入到对话上下文

**期望**：
- 支持拖拽/点击上传 PDF/图片/代码文件
- 上传后显示缩略图/文件名 tag
- 发送时将文件内容（或 base64 图片）注入到 API 请求

### P3：模型选择硬编码（图六-第一张 + 第二张）
**现象**：
- 底部显示 `gpt-4o-mini` 但这只是硬编码默认值
- 点击模型名称无法展开真实模型列表
- 无法根据已配置的 API Key 实时查询可用模型并切换

**期望**：
- 点击模型名称 → 弹出下拉列表 → 显示从 API 实时获取的模型
- 支持搜索/筛选模型
- 切换后立即生效

### P4：Fast/Plan 模式无差异（图六-第一张）
**现象**：
- 右上角有 `Fast` 和 `Plan` 按钮可切换
- 切换后除了按钮高亮变化外，AI 行为、系统提示、工具可用性完全相同
- Context 面板显示「规划模式」但没有实际的规划行为差异

**期望**：
- **Fast 模式**：直接回答，最少工具调用，快速响应
- **Plan 模式**：先输出思考步骤/计划，逐步执行，展示推理过程

### P5：输入框无法选中/输入（第三批反馈-图四）
**现象**：对话输入框显示"请先配置 API Key..."的 placeholder，但鼠标点击后无法获得焦点、无法输入任何文字。
**根因**：可能 `disabled` 条件判断有误或 CSS `pointer-events: none` 阻断了交互。即使未配置 Key 也应允许输入（发送时再提示）。

### P6：Token 计数虚假（第三批反馈-图三）
**现象**：底部状态栏持续显示 `Token: 0 / 128.0k (0%)`，数值从未变化过。实际发送消息后 Token 没有增减。
**根因**：AgentLoop 未接入 Token 统计回调，状态栏的数值是硬编码初始值。

### P7：连接状态虚假（第三批反馈-图三）
**现象**：底部状态栏始终显示绿色 `● 已连接`，即使 API Key 未配置或网络不通。
**根因**：连接状态是硬编码 `true`，未做真实的 API 可达性检测。

### P8：模型选择器不可用（第三批反馈-图一+图二）
**现象**：
- 底部显示 `✦ gpt-4o-mini` 是硬编码默认值
- 点击模型名称无任何响应，不展开模型列表
- 即使在设置里获取了模型列表，对话面板的模型选择器不联动
- 设置里「默认模型」下拉也是空的、「获取模型」按钮存在但结果未持久化显示

## 目标

### 核心通信闭环
1. API Key 从设置 → Redux → AIClient 完整传递链验证
2. Streaming 响应实时渲染到对话区
3. 错误信息清晰展示（网络/鉴权/限流等）
4. 工具调用 → 执行 → 结果回传 → 继续对话 全闭环

### 文件上传
1. 拖拽 + 点击上传
2. 支持主流格式：图片(jpg/png/gif)、文档(pdf/docx/md/txt)、代码文件
3. 上传后显示缩略图/文件 tag
4. 图片 → base64 注入 `content` (vision API)
5. 文档 → 提取文本注入上下文

### 模型选择器
1. 底部模型名称可点击展开
2. 首次打开 / API Key 变更时自动 fetchModels
3. 模型列表支持搜索
4. 切换即生效

### 模式差异
1. Fast → systemPrompt 精简 + 禁用部分工具 + temperature 略高
2. Plan → systemPrompt 含推理指令 + 全工具开放 + 显示 thinking 块
